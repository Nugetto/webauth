Assignment Report

Introduction

In this assessment for Web Authoring we were tasked with creating a website of a band or brand using our newly-learned skills of HTML, CSS, and JavaScript. The website had to comply with current web standards. This document explains the process of the creation of the website, ideas, about the website itself, what skills I have learned and been able implement, and my strengths and weaknesses.

Structure

I have taken effort to organise my folders within my website root so that it is easier to reference them and so the website is more efficient. From Figure 1 you can see that I have my main HTML documents in the root folder, this is because I don’t have many pages so this makes it simpler to reference them. If I were to have a lot more HTML files then I would go with a more hierarchical structure to organise the HTML pages into separate directories. I also have separate folders for my assets (Images, sounds, videos, etc), a folder for my CSS files and another for my JS files. Keeping these separate helps me find them and reference them in my code easier and quicker without the hassle of going through many files.

Current Web Standards

In this section I am going to be talking about the strengths and weaknesses of current web standards. The first strength of current web standards means it is more easily accessible to everybody who has access to the internet. With advances of technology such as text-to-speech which allows the blind to access the internet, this is because each image must have an alt tag which states what the image is of as of the current web standards. Another advantage is that there are many websites that validate your code for you, allowing you to easily understand how to make your website ready for current web standards. One weakness would be that not all browsers support certain functions, meaning research has to be done and further code has to be implemented to make sure it is able to run on all browsers (or at least most). This can take a lot longer than it would do otherwise, meaning time constraints is a weakness. The website has been tested on multiple browsers to make sure it is at a good enough standard. Luckily enough, all of the code I have used to create my website works on all browsers.

Techniques Used

I have used different techniques in my jQuery, JS and CSS to make sure my website is professional and interactive. I used local storage in my website to save data and then use this data again if the user closes or refreshes the webpage. Specifically, the user presses buttons to select certain music beats and each time they press one, their selected buttons are saved and then loaded again when back on the page if they leave. Another technique used is a slider for my customiser which allows them to change the tempo of the beat. The slider ranges from 100 (0.1s) to 1000 (1s). I have also implemented the feature of audio to my website as the customiser is about making your own song/beat.

Challenges

One of the main challenges that I faced during this assessment was designing my website so it could be used on mobile as well as on a desktop. I built my website desktop first, meaning I had to then redesign it for mobile which became tricky. In the end I had to remove certain elements on mobile to overcome the obstacles of not being able to get them to work. I used relative values for sizes and distances and also used CSS media queries for specific sized windows. Another challenge I had was saving what buttons the user has pressed on the customiser. I eventually concluded having a large binary string, with each button have an on and off state, either being a 1 or 0. This worked well and was saved to local storage every time a button was pressed.
